# День 6. Автоматизація і агенти

> **Сьогоднішні ролі.** Пілот — учорашній Навігатор. Сьогодні Навігатор перевіряє безпеку: жодних API-токенів у код-комітах, жодних credential-файлів у спільних папках. Помилка тут — вразливість на роки вперед.

### Сцена
Досі ти використовував AI як консультанта. Сьогодні перетворюєш його на співробітника. AI з руками — той, що сам робить речі, а не лише радить, як їх зробити.

n8n — автоматизація між сервісами. Claude Code — агент, що працює з твоїми файлами. MCP — протокол, що з'єднує AI з твоїм Notion, Drive, GitHub. Сьогоднішнє питання: яка рутинна задача може більше не існувати у твоєму житті?

### Лекційна частина

**Чому автоматизація — найбільший важіль у роботі**
Принцип Парето: 80% результату йде від 20% дій. Інші 80% часу — це повторювана робота, яку можна автоматизувати.

Класичний експеримент: спитай людину "скільки часу ти витрачаєш на email на день?". Відповідь: "15-20 хв". Реально (з трекерів): 2-3 години. Більшість роботи — невидима повторювана рутина.

Автоматизація = перетворити одну "ручну" дію на одну налаштовану систему, що працює сама. Економія часу за рік: 50-200 годин для типового підприємця/працівника.

**xkcd "Is It Worth The Time?" — формула**

Класична таблиця з xkcd: скільки часу витратити на автоматизацію залежить від:
- Як часто завдання повторюється
- Скільки часу займає кожен раз
- Як часто треба буде повторити

Приклад: якщо завдання займає 5 хв і робиться щодня — автоматизація варта до 6 годин зусиль (за 5 років економить більше).
Якщо завдання займає 5 хв і робиться раз на місяць — варта 5 хв зусиль (тобто не варта).

Правило великого пальця: автоматизуй, якщо завдання повторюється щотижня або частіше.

**Workflow design thinking**
Перш ніж будувати, постав 4 питання:
1. *Що запускає завдання?* (trigger) — час, подія, ручний натиск
2. *Що треба зробити?* (action) — серія кроків
3. *Які умови?* (conditions) — "якщо так — роби це, інакше — те"
4. *Куди йде результат?* (output) — де я побачу, що зробилось

Більшість багів автоматизації — від нечіткого формулювання цих 4 моментів.

**Triggers / Actions / Conditions — базова логіка**

*Triggers (запуски):*
- *Time-based:* "Щодня о 09:00", "Щопонеділка"
- *Event-based:* "Коли надходить email від X", "Коли заповнили форму", "Коли додали рядок у Sheets"
- *Webhook:* "Коли інший сервіс послав сигнал"
- *Manual:* "Коли я натиснув кнопку"

*Actions (дії):*
- Зробити HTTP-запит
- Створити/оновити запис у БД (Sheets, Airtable, Notion)
- Відправити email/SMS/Telegram
- Зробити AI-генерацію (текст, зображення)
- Викликати інший workflow

*Conditions (умови):*
- IF/ELSE: "якщо email від [важливий клієнт] — пінгувати в Telegram, інакше — звичайна папка"
- Switch: різні гілки залежно від типу даних
- Filter: пропускати дані тільки якщо вони відповідають критеріям

**Webhook як універсальний інтерфейс**
Webhook — це URL, на який можна послати дані, і "хтось" їх отримає і обробить. Технічно це POST-запит з JSON-тілом на цей URL.

Чому це універсальний клей:
- Будь-який сервіс зараз вміє надсилати webhook після події (нова реєстрація, новий лід, нова покупка)
- Будь-який сервіс зараз вміє приймати webhook як тригер
- n8n, Make, Zapier — це фактично "приймачі webhook → процес → відправка наступному"

Якщо у сервісу немає прямого "ноду" в n8n — використай HTTP Request node і генеричний webhook.

**APIs: коли треба глибше за webhook**
API (Application Programming Interface) — спосіб двох систем "говорити" одна з одною через структуровані запити.

Webhook — це односторонній сигнал ("щось сталось"). API — це діалог ("дай мені список останніх замовлень", "створи новий контакт у CRM").

Більшість сучасних інструментів мають REST API. Документація завжди починається з:
- *Endpoint:* URL, на який слати запит
- *Method:* GET (отримати), POST (створити), PUT (оновити), DELETE (видалити)
- *Auth:* API key, OAuth, Bearer token
- *Request body:* JSON з параметрами
- *Response:* що повернеться

n8n має HTTP Request node для будь-якого API. Claude Code може писати запити на API. Це фундамент сучасної інтеграції.

**MCP (Model Context Protocol): як AI отримує доступ до твоїх даних**
MCP — стандарт, розроблений Anthropic у 2024, для з'єднання AI-моделей з зовнішніми системами. Відкритий протокол, який підтримують вже OpenAI, Google і ще десятки сервісів.

Раніше: щоб Claude міг працювати з твоїм Notion, треба було "копіювати-вставляти" вміст у чат.
З MCP: ти даєш Claude доступ до Notion через MCP-сервер, і Claude може читати, шукати, оновлювати твої сторінки прямо.

MCP-сервери є для:
- Notion, Linear, Jira, Asana (управління завдань)
- Google Drive, Gmail, Calendar
- GitHub, GitLab
- Slack, Discord
- Spotify, навіть Home Assistant

Конфігурація: у Claude Desktop або Claude Code додаєш у settings.json URL MCP-сервера. Claude автоматично виявляє доступні інструменти.

**Claude Code як агент: парадигма "AI з руками"**
Класичний Claude — це чат. Ти питаєш, він відповідає. Claude Code — це AI, який має "руки": може запускати команди в терміналі, читати/писати файли, робити git-операції.

Це не "AI пише код за тебе". Це "AI виконує задачу, в якій код — це інструмент". Приклади задач для Claude Code:
- "Об'єднай усі мої research-PDFs у один Notion-документ з summary"
- "Згенеруй 50 варіацій pitch-листа з моєї таблиці клієнтів"
- "Зроби бекап усіх моїх Google Docs у local папку"
- "Розшифруй цей mp3-файл у Whisper і збережи як текст"

Це задачі, які роблять "девелопер-аналітики". З Claude Code — їх роблять не-програмісти.

**n8n vs Make vs Zapier: коли який**

*Zapier* — найстарший, найпростіший, найдорожчий. 6000+ інтеграцій. Безкоштовно — 100 задач/місяць.

*Make (раніше Integromat)* — складніший інтерфейс (візуальні діаграми), більше гнучкості, кращі ціни. Має ту саму кількість інтеграцій.

*n8n* — open-source, можна self-host (на твоєму сервері = безкоштовно). Менше готових інтеграцій, але є HTTP Request для будь-чого. Технічно складніший, але набагато гнучкіший.

Правило вибору: разові задачі + не цінуєш гнучкість → Zapier. Серйозний бізнес з постійними автоматизаціями → Make. Технічний контроль і безкоштовно → n8n.

**Безпека в автоматизації: 3 правила**
1. *Принцип мінімального доступу:* кожен токен/API key має тільки потрібні права. Не давай "admin" коли вистачить "read".
2. *Не зберігай credentials у клер-тексті:* у n8n credentials шифруються, але якщо self-hosting — налаштуй це.
3. *Логи:* кожна автоматизація має лог. Якщо щось зламалось — мусиш бачити, де і коли.

Найбільший ризик: webhook без верифікації. Хтось може послати на твій webhook фейкові дані і запустити дії. Завжди додавай secret token у webhook URL.

**n8n: cloud vs self-host**
- *Cloud (n8n.io):* найшвидший старт, безкоштовно 14 днів trial, потім $24/міс. Не треба нічого налаштовувати

- *Self-host локально (Docker):* для досвідчених. `docker run -p 5678:5678 n8nio/n8n`

Для практики — Cloud простіше.

**n8n: перший workflow крок за кроком (приклад "форма реєстрації")**

*Крок 1: Створи workflow*
- n8n Cloud → My Workflows → "+ Add workflow"
- Дай назву: `[Company Name] — Registration Flow`

*Крок 2: Тригер — форма*
- "+" → Trigger → On Form Submission (вбудована форма n8n)
- Form Title: `Реєстрація на подію [Company Name]`
- Налаштуй поля:
    - Name (Text, required)
    - Email (Email, required)
    - Phone (Number, optional)
    - Message (Text long, optional)
- Кнопка "Save" → отримаєш Production URL форми. Скопіюй.

*Крок 3: Запис у Google Sheets*
- "+" з виходу тригера → "Google Sheets" → "Append or Update Row"
- "+ Add Credential" → OAuth Google → дозволь доступ
- Document: вибери свій Sheet з dropdown
- Sheet: "Sheet1" (або як назвав)
- Operation: "Append"
- Mapping (drag з лівої панелі або вручну):
    - Column "Name": `{{ $json.Name }}`
    - Column "Email": `{{ $json.Email }}`
    - Column "Phone": `{{ $json.Phone }}`
    - Column "Message": `{{ $json.Message }}`
    - Column "Timestamp": `{{ $now }}`

*Крок 4: Лист підтвердження (Gmail)*
- "+" → Gmail → "Send a Message"
- "+ Add Credential" → OAuth Google → дозволь
- To: `{{ $json.Email }}`
- Subject: `Дякуємо за реєстрацію на [Company Name]!`
- Message Type: HTML
- Body:
```html
<p>Привіт, {{ $json.Name }}!</p>
<p>Дякуємо за реєстрацію. Незабаром ми надішлемо деталі події.</p>
<p>Якщо маєш питання — напиши у відповідь.</p>
<p>До зустрічі!</p>
```

*Крок 5: Пінг у Telegram*

Спочатку — створи бота:
1. Відкрий @BotFather у Telegram → `/newbot`
2. Дай назву бота (наприклад "Registrations Bot")
3. Дай username (закінчується на "_bot", наприклад "company_registrations_bot")
4. Збережи token (виглядає як `7234567890:AAH...`)

Потім — отримай chat_id:
1. Створи групу або канал у Telegram, додай туди свого бота
2. Напиши будь-яке повідомлення у групу
3. У браузері відкрий: `https://api.telegram.org/bot[ТВІЙ_TOKEN]/getUpdates`
4. Знайди `"chat":{"id":-123456789...}` — це chat_id (може бути негативним для груп)

Тепер у n8n:
- "+" → Telegram → "Send a Text Message"
- "+ Add Credential" → встав token бота
- Chat ID: твій chat_id
- Text:
```
🔔 Нова реєстрація!

👤 {{ $json.Name }}
📧 {{ $json.Email }}
📱 {{ $json.Phone }}
💬 {{ $json.Message }}

🕐 {{ $now.format('dd.MM.yyyy HH:mm') }}
```

    - Чи з'явився рядок у Google Sheets?

**n8n: додавання логіки (Conditions, IF/ELSE)**
Базовий workflow вище — лінійний. Додамо логіку: якщо в формі вказано "VIP" — пінгувати в окремий канал.

1. Між Sheets і Gmail встав вузол: "+" → Flow → "IF"
2. Condition: `{{ $json.Message }}` contains `VIP`
3. True гілка → новий Telegram-вузол → інший chat_id (для VIP-каналу)
4. False гілка → продовжує до Gmail звичайного

Це базовий патерн розгалуження. Так само можна:
- Перевіряти, чи email з конкретного домену
- Фільтрувати за регіоном (якщо є поле "місто")
- Розгалуджувати за типом запиту

**n8n: типові патерни автоматизації (для натхнення)**

*Контент-конвеєр:*
- Trigger: новий рядок у Google Sheet (контент-план)
- Action 1: AI generation (через HTTP node до Claude API) — згенерувати повний пост
- Action 2: збереження як Notion-сторінка
- Action 3: scheduled посилання у Buffer/Later API для публікації

*Email-summarizer:*
- Trigger: новий email у Gmail з міткою "review-later"
- Action: HTTP до Claude API з промптом "Зроби короткий summary і визнач пріоритет"
- Action: створити запис у Notion з summary + priority + посилання на оригінал

*Daily digest:*
- Trigger: cron, щодня о 09:00
- Action: пройти по 5 джерелах (RSS, Twitter API, Reddit)
- Action: збагатити через Claude API (summary + класифікація)
- Action: відправити в Telegram-канал

**Claude Code як агент: повний workflow (особистий трек)**

**MCP-сервери: підключення до зовнішніх сервісів**

MCP (Model Context Protocol) дозволяє Claude мати "руки" в Notion, Google Drive, GitHub, etc. Установка проста:

1. У Claude Code: `claude mcp` → побачиш список доступних серверів
2. Додай Notion: `claude mcp add notion`
3. Потрібен буде API token з Notion → Settings → Integrations → New integration
4. Скопіюй token, встав у Claude Code

Після цього можеш давати команди як: "Створи нову Notion-сторінку у БД 'Tasks' з заголовком X і описом Y". Claude сам зробить це через MCP.

Найкорисніші MCP-сервери для практики:
- *Notion* — створення/читання сторінок
- *Google Drive* — операції з файлами
- *Slack* — повідомлення в канали
- *GitHub* — управління репозиторіями
- *Linear / Jira* — створення тікетів

**Найкращі практики**
- **Перший workflow — найпростіший.** Один тригер → одна дія. Не намагайся збудувати все за раз.
- **Test з реальними даними швидко.** Не плануй workflow годинами — план + тест за 20 хв.
- **Безпека для Claude Code.** Не запускай у папках з паролями (.env, ssh ключі). Створи окрему workspace папку.
- **Webhook як універсальний інтерфейс.** Будь-який сервіс, що приймає webhook, можна під'єднати до n8n. Якщо немає прямого ноду — generic HTTP Request.
- **Документуй автоматизації.** Через 3 місяці ти не пам'ятатимеш, що цей workflow робить. Notion-сторінка з описом для кожного workflow — обов'язково.
- **Один workflow = одна мета.** Не намагайся зробити "універсальний автоматизатор всього". Краще 5 окремих простих workflow, ніж один складний.
- **Error handling — критично.** У n8n додавай "Error Trigger" вузол з пінгом у Telegram, коли щось ламається. Без цього workflow може мовчки померти.
- **Версіонуй workflow.** n8n дозволяє експорт як JSON. Тримай у GitHub-репо — це твоя production-документація.
